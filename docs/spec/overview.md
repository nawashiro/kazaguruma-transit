# 基本仕様

## 目的

この WEB アプリケーションは千代田区福祉交通「風ぐるま」の乗換案内を提供します。「風ぐるま」は障害者を含むあらゆる属性の人に福祉施設へのアクセスを提供しています。GoogleMap などは健常者向けの交通機関などのノイズが多いのに対し、本アプリケーションは「風ぐるま」利用に対して最もシンプルな手段を提供します。

## 設計思想

1. 本当に必要なものをわずかに提供します。目的地、出発地、時刻などの必要パラメータを入力されたら、ただひとつの最適経路のみを返却します。
2. 障害者や高齢者へ配慮が必要です。HTML、WCAG2.1 レベル AA、WAI-ARIA の原則を確実に守る必要があります。
3. Docker コンテナは状態を持たず、永続的なデータを扱いません。GTFS データを Sqlite に格納しますが、あくまでキャッシュであって、失われることを前提としています。

参考資料:

- https://www.w3.org/TR/WCAG21/
- https://developer.mozilla.org/ja/docs/Learn_web_development/Core/Accessibility/WAI-ARIA_basics

## 開発方針

テスト駆動開発を原則とします。しかし、完全なテストは存在せず、それはコストとトレードオフします。テストやモックの実装が広範にわたるなど、比較的高コストな場合は省略します。

変数や API レスポンス、フォームなどは any を避け、型定義を行います。共通で使用する定義を抽出し、一元管理し、ドキュメントを残す必要があります。
