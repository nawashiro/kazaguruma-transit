# 検証ガイド: UI 最小フォントサイズ準拠

## 前提条件

- 依存パッケージをインストール済みであること。
- ローカル設定と、画面表示に必要な既存データが利用可能であること。

## 自動検証

1. 実装前に、通常UIの 14px 未満のサイズ指定を検出するテストを追加する。このテストが既存の
   違反箇所で失敗することを確認する。
2. 違反箇所を是正し、次を実行する。

   ```bash
   npm run lint
   npm test
   npm run build
   ```

3. すべて成功し、サイズ監査テストがルビ補助テキストだけを例外として扱うことを確認する。

## 手動表示確認

1. 開発サーバーを起動し、`screen-state-matrix.md` に列挙された全ルートと共有UIを表示する。
2. 各画面で、初期・通常、読み込み、空、エラー、操作後通知、開閉UIのうち実装されている状態を確認する。
3. 1280pxと390pxの表示幅で、対象テキスト要素のブラウザ上の算出 `font-size` を確認し、台帳へ
   `PASS` または `FAIL (path:line)` を記録する。
4. ルビ付き表示では本文が 14px 以上であり、14px 未満なのは `rt` の補助テキストだけであることを
   確認する。

期待結果は [UI文字サイズ準拠契約](./contracts/font-size-compliance.md) の不変条件をすべて満たすこと。
