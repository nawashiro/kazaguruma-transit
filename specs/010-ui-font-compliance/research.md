# 調査記録: UI 最小フォントサイズ準拠

## 判断 1: 14px の標準ユーティリティを下限として使う

- **Decision**: 通常のユーザー向けテキストは 14px 相当以上のサイズを使い、12px 相当の
  小サイズ指定を通常UIから取り除く。
- **Rationale**: リポジトリの Tailwind CSS 4 定義では `text-sm` が `0.875rem`（既定の
  16px ルートサイズで 14px）、`text-xs` が `0.75rem`（12px）である。現行コードには
  `text-xs` のユーザー向け使用箇所があり、憲章に違反する。
- **Alternatives considered**:
  - すべての文字を一律に拡大する: 見出し・本文・補助情報の既存の階層を不必要に変えるため不採用。
  - `text-xs` を許可リスト化する: ルビ以外の例外を作り、憲章の明確な下限に反するため不採用。

## 判断 2: ルビの `rt` だけを例外として維持する

- **Decision**: `rt` の補助テキストは現行のルビ用スタイルを維持し、ルビ本文は通常の下限に従う。
- **Rationale**: 憲章が例外として認めるのはルビの補助テキストのみであり、本文の可読性を損なわずに
  読み補助を提供できる。
- **Alternatives considered**:
  - ルビを無効化する: 読み補助機能を失うため不採用。
  - ルビ本文も例外とする: 例外の範囲を不当に広げるため不採用。

## 判断 3: 自動監査と表示確認を併用する

- **Decision**: 14px 未満の通常UI用サイズ指定を検出する自動テストを先に作成し、広い・狭い画面幅の
  手動確認で最終判断する。
- **Rationale**: 自動監査は再発を防ぎ、実表示の確認はレスポンシブ条件・コンポーネントライブラリ由来の
  表示を補完する。
- **Alternatives considered**:
  - 手動確認だけ: 今後の変更で見落としやすいため不採用。
  - 静的検査だけ: 表示幅に応じた実際の見え方を保証できないため不採用。
