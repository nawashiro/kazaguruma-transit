# 画面・状態検証台帳

この台帳は、010仕様における「全画面・全表示状態」の対象を固定する。各対象を
広い表示幅（1280px）と狭い表示幅（390px）で確認し、対象テキストの算出
`font-size` が14px以上であることを記録する。

## 判定ルール

- 対象状態は、初期表示、成功・通常表示、読み込み、空、エラー、操作後通知、開閉UIのうち、画面が提供するものを確認する。
- 存在しない状態は「N/A」とし、対象外理由を備考に記録する。
- `rt` の補助テキスト、アイコンのみの操作、画像内文字、PDF出力、第三者コンテンツは対象外とする。
- `sr-only` など利用者に表示されない情報は、算出値確認の対象外とする。

## 記録

各セルは `未確認`、`PASS`、`FAIL (path:line)` のいずれかに更新する。FAILの場合は、
対象要素、算出値、表示幅、状態、修正対象ファイルを備考または課題に記録する。

## 2026-07-11 実測記録

- Puppeteerで全15ルートを1280pxと390pxで描画し、30ケースを確認した。
- 全ケースがHTTP 200で、表示テキストの算出 `font-size` が14px未満となる要素は0件だった。
- 静的監査テストで、全ソースの小サイズユーティリティとCSS指定、および `rt` 以外の例外を確認した。
- 状態列は、状態を持つ実装箇所を静的監査の対象に含め、動的状態の最終確認は各画面の実データ・操作条件が利用可能な環境で再実行する。
