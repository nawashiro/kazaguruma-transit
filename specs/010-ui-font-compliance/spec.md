# Feature Specification: UI 最小フォントサイズ準拠

**Feature Branch**: `010-ui-font-compliance`

**Created**: 2026-07-11

**Status**: Draft

**Input**: User description: "UIが憲章に従うようにしてください。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 通常テキストを読み取る (Priority: P1)

利用者は、画面上の案内、操作ラベル、入力補助、状態表示を無理なく読み取れる。

**Why this priority**: 小さすぎる文字は、経路検索や議論などの主要な利用を直接妨げるため。

**Independent Test**: 各ユーザー向け画面を確認し、ルビ以外の表示テキストが 14px 以上であることを検証できる。

**Acceptance Scenarios**:

1. **Given** 利用者が任意の画面を表示している、**When** 案内文、ラベル、ボタン、リンク、入力補助、状態表示を確認する、**Then** ルビ以外のすべてのユーザー向けテキストが 14px 以上で表示される。
2. **Given** 利用者が狭い画面幅で画面を表示している、**When** 画面幅に応じた表示を確認する、**Then** テキストが縮小されて 14px 未満にならない。

---

### User Story 2 - 一時的な表示を読み取る (Priority: P2)

利用者は、開閉する画面要素や処理結果のメッセージも他の画面と同じ最低文字サイズで読み取れる。

**Why this priority**: エラーや操作結果を読めないと、利用者は次に何をすべきか判断できないため。

**Independent Test**: ダイアログ、メニュー、ツールチップ、通知、空状態、エラー状態を表示し、各テキストサイズを検証できる。

**Acceptance Scenarios**:

1. **Given** 利用者が操作により一時的なUIを表示する、**When** その内容を確認する、**Then** ルビ以外のユーザー向けテキストは 14px 以上で表示される。
2. **Given** 利用者にエラーまたは空状態が表示される、**When** メッセージと次の操作を確認する、**Then** どちらも 14px 以上で表示される。

---

### User Story 3 - ルビ付きの日本語を読む (Priority: P3)

利用者は、本文を十分な大きさで読みつつ、ルビを補助情報として確認できる。

**Why this priority**: 日本語の読み補助を維持しながら、憲章上の例外範囲を明確にするため。

**Independent Test**: ルビを含む表示を確認し、本文が 14px 以上であり、例外がルビの補助テキストだけに限られることを検証できる。

**Acceptance Scenarios**:

1. **Given** ルビを含むテキストが表示されている、**When** 本文と補助テキストを確認する、**Then** 本文は 14px 以上であり、14px 未満を許容するのはルビの補助テキストのみである。

### Edge Cases

- 画面幅、文字列の長さ、折り返しの有無によって、通常テキストが 14px 未満に縮小されない。
- アイコンだけの操作、画像内の文字、利用者に表示されない補助情報は、本機能の文字サイズ確認対象外とする。
- 開発用画面または第三者コンテンツで、アプリケーションが文字サイズを制御できないものは対象外とし、利用者向けのアプリケーションUIに混在させない。

### Verification Definitions

- 「14px以上」は、対象テキスト要素に適用されたCSSの算出 `font-size` が14px以上であることを指す。Tailwind/DaisyUIのクラス名、`rem`、`%`、継承値、メディアクエリおよび動的なクラス切り替えは、最終的な算出値で判定する。
- 自動監査は静的な小サイズ指定を早期検出するために使用し、算出値の保証には対象画面をブラウザで描画して確認する検証を併用する。
- 対象画面・表示状態・表示幅は `screen-state-matrix.md` に列挙する。対象を「全画面」と呼ぶ場合、その台帳に列挙されたすべてのルートと状態を意味する。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、利用者に表示するすべてのUIテキストについて、ルビの補助テキストを除き、算出 `font-size` を 14px 以上にしなければならない。
- **FR-002**: システムは、画面幅、レスポンシブ指定、継承、動的な表示状態の違いによって、FR-001 の対象テキストを 14px 未満にしてはならない。
- **FR-003**: システムは、ページ本体に加え、ナビゲーション、フォーム、ボタン、リンク、ダイアログ、メニュー、ツールチップ、通知、空状態、読み込み状態、エラー状態のユーザー向けテキストに FR-001 を適用しなければならない。
- **FR-004**: システムは、ルビ付きテキストの本文を 14px 以上で表示しなければならない。14px 未満を許容できるのはルビの補助テキストだけである。
- **FR-005**: システムは、今後追加または変更するユーザー向けUIについても、公開前に FR-001 から FR-004 への適合を確認可能にしなければならない。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: `screen-state-matrix.md` に列挙した全画面および全表示状態の確認で、ルビの補助テキスト以外に 14px 未満のユーザー向けテキストが 0 件である。
- **SC-002**: `screen-state-matrix.md` に定義した狭い画面幅と広い画面幅の両方で、確認対象テキストの 100% の算出 `font-size` が 14px 以上である。
- **SC-003**: ルビを含む確認対象の 100% で、本文が 14px 以上であり、14px 未満のテキストがルビの補助テキストだけである。
- **SC-004**: 対象画面のエラー、読み込み、空状態および操作後の通知を確認したとき、14px 未満のユーザー向けテキストが 0 件である。

## Assumptions

- 対象は、アプリケーションが管理する利用者向けの画面と一時的なUIである。
- ルビの補助テキストとは、日本語本文の読みを補助するために本文に付随するテキストだけを指す。
- 外部サイト内の表示、画像へ焼き込まれた文字、利用者に表示されない情報は対象外である。
- 既存の画面構成と利用フローは維持し、文字サイズ準拠に必要な範囲だけを変更する。
