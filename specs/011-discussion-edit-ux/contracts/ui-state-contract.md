# UI State Contract: 会話関連タブと編集画面

## 共通タブ

会話関連ページは、同一の会話識別子に対して次の3タブを表示する。

| 表示名 | 遷移先 | 選択条件 |
|---|---|---|
| 会話 | `/discussions/{naddr}` | パスが会話詳細 |
| 監査ログ | `/discussions/{naddr}/audit` | パスが監査ログ |
| 編集 | `/discussions/{naddr}/edit` | パスが編集 |

- 3タブはすべての利用者に表示する。
- 現在のタブのみ `aria-selected="true"` と視覚的な選択状態を持つ。
- 非選択タブもキーボードでフォーカス・移動できる。
- 「会話に戻る」専用リンクは編集ページに置かない。
- 各タブの操作領域は44px以上を確保する。

## 権限表示

未ログイン時に認証が必要な操作群を表示する場合、次を同じ文脈に置く。

1. 操作できない理由（例: 「編集操作にはログインが必要です。」）。
2. 「ログイン」ボタン1つ。
3. ログイン成功後に元の編集ページへ戻れること。

ログイン済み非作成者には、未ログイン用のログイン案内を表示せず、「会話作成者のみ編集できます。」などの権限理由を表示する。

## 取得状態

| 状態 | 主表示 | 利用可能な操作 |
|---|---|---|
| loading | 会話情報を読み込み中 | 一覧・タブによる離脱 |
| delayed | 取得遅延の警告 | 再読み込み・再試行 |
| success | 会話情報と編集フォーム | 権限に応じた操作 |
| not-found | 会話が見つからない説明 | 会話一覧へ戻る |
| error | 取得失敗の説明 | 再試行 |

`loading` 中に `not-found` の文言を表示してはならない。

## 編集ページ構造

```text
会話情報を編集
├── 会話情報編集
├── 掲載申請
├── モデレーター管理
└── 危険な操作
    └── 会話を削除 -> 確認ダイアログ
```

各グループの権限説明は重複させず、操作群の目的と状態を明確にする。主要操作はモバイル幅で縦積みまたは折り返し、横方向に画面外へ出してはならない。

## 互換性

- 既存のNostrイベント、認証、削除確認、ルビ設定の永続化形式を変更しない。
- 画面上の日本語文言は既存のルビ表示ユーティリティと共存する。
- 支援技術向けには単一のmainランドマークと論理的な見出し階層を提供する。
