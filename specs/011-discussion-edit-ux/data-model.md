# Data Model: 会話編集画面UX改善

本機能では新しい永続データを追加しない。既存の会話データと認証情報を、表示・操作制御のために次の状態へ整理する。

## 既存の編集対象会話

| 属性 | 用途 | 制約 |
|---|---|---|
| `title` | 編集フォームのタイトル | 必須、100文字以内 |
| `description` | 編集フォームの説明 | 必須、500文字以内 |
| `dTag` | 読み取り専用の会話識別子 | 編集不可 |
| `authorPubkey` | 編集権限の判定 | 現行の作成者判定を維持 |
| `moderators` | 現在のモデレーター表示と申請判定 | 現行のNostrイベント由来データを維持 |
| `event` | 保存・削除・モデレーションイベントの基礎情報 | 既存イベント形式を維持 |

## 利用者権限状態

```text
unauthenticated
authenticated-non-author
author
current-moderator
author-and-moderator
```

権限状態は既存の `user` と会話の `authorPubkey`・`moderators` の関係から導出する。UI改善によって権限の範囲を変更しない。

| 状態 | 編集 | 掲載申請 | 削除 | 昇格申請 | 昇格審査 |
|---|---:|---:|---:|---:|---:|
| 未ログイン | 不可 | 不可 | 不可 | 不可 | 不可 |
| ログイン済み非作成者 | 不可 | 不可 | 不可 | 条件付き可 | 不可 |
| 作成者 | 可 | 可 | 可 | 不可 | 可 |
| 現在のモデレーター | 不可 | 不可 | 不可 | 不可 | 不可 |
| 作成者兼モデレーター | 可 | 可 | 可 | 不可 | 可 |

## 会話取得状態

```text
loading -> success
loading -> delayed
loading -> not-found
loading -> error
delayed -> loading (retry)
error -> loading (retry)
```

- `loading`: 会話データの取得が完了していない。不存在表示を出さない。
- `success`: 会話データが存在し、編集フォームを表示する。
- `delayed`: relay応答遅延またはタイムアウト。再試行案内を表示する。
- `not-found`: 取得処理が完了し、会話データが存在しないことが確認された。
- `error`: 取得失敗。エラー説明と再試行を表示する。

## 編集操作グループ

1. **会話情報編集**: タイトル、説明、モデレーター入力、保存。
2. **掲載申請**: 会話一覧への掲載申請。
3. **モデレーター管理**: 現在のモデレーター、昇格申請、申請一覧、作成者による審査。
4. **危険な操作**: 削除。確認ダイアログを経由する。

## データ整合性

- Nostrイベントの形式、relayを正本とする方針、既存の認可判定は変更しない。
- ログインモーダルの開閉や再試行によって、取得済みのフォーム値を意図せず初期化しない。
- 取得状態が切り替わる際、古い会話情報と不存在・遅延表示を同時に見せない。
