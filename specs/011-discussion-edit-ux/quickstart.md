# Quickstart: 会話編集画面UX改善の検証

## 前提

- リポジトリルートで作業する。
- Node.js依存関係がインストール済みである。
- ローカル開発サーバーを起動できる。
- 指定の会話naddrを用いた編集URLを利用する。

## 自動テスト

```bash
npm test -- --runInBand \
  src/components/discussion/__tests__/DiscussionTabLayout.test.tsx \
  'src/app/discussions/[naddr]/edit/__tests__/page.test.tsx'
```

確認項目:

- タブが「会話」「監査ログ」「編集」の3件である。
- 詳細・監査ログ・編集の各パスで選択状態が正しい。
- キーボードの左右、Home、Endで3タブを移動できる。
- 未ログイン時に権限メッセージとログインボタンが表示される。
- 削除が危険な操作セクションにあり、確認操作を経由する。
- loading、遅延、取得成功、不存在、エラーが混同されない。

## 静的検証

```bash
npm run lint
npm test -- --runInBand
npm run build
```

## ブラウザ検証

1. `npm run dev` を起動する。
2. 指定された `/discussions/{naddr}/edit` を開く。
3. 幅1440pxで、3タブ、編集フォーム、モデレーター管理、危険な操作を確認する。
4. 幅390pxで、横スクロールが発生しないことを確認する。
5. 幅320pxでも、タイトル・説明・主要ボタン・削除操作が画面幅内に収まることを確認する。
6. ルビ表示をオン・オフし、固定ルビコントロールが入力欄、リサイズハンドル、主要操作を覆わないことを確認する。
7. 未ログイン状態で「編集」タブを開き、権限メッセージ内の「ログイン」ボタンから認証モーダルを開く。
8. 会話タブ、監査ログタブ、編集タブを移動し、「会話に戻る」専用リンクが表示されないことを確認する。
9. 会話取得を遅延させた状態で、読み込み中に「会話が見つかりません」が表示されないことを確認する。
10. 未ログイン状態から編集タブを開き、検証者が1回以内の操作で権限メッセージ内の「ログイン」ボタンへ到達できるかを記録する。
11. タイトル・説明の編集、保存操作の確認を、幅320px・390px・1440pxで複数の検証者が初回操作で完了できる割合を記録する。

## 合格条件

- 幅320px以上で横方向オーバーフローがない。
- 主要操作が画面内で押せる。
- 固定ルビコントロールが重要UIを覆わない。
- 3タブの選択状態とキーボード移動が正しい。
- 権限別の表示と既存の認可動作が一致する。
- 未ログイン導線の到達率が100%、主要編集シナリオの初回完了率が90%以上である。
- `npm run lint`、対象テスト、全テスト、`npm run build` が成功する。
