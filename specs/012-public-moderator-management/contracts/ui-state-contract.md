# UI State Contract: 公開モデレーター管理

## ルートと可視性

| 画面 | 会話作成者 | ログイン済み一般ユーザー | 未ログインユーザー |
|---|---|---|---|
| 会話 | 表示 | 表示 | 表示 |
| 監査ログ | 表示 | 表示 | 表示 |
| モデレーター | 公開一覧と変更選択UIを表示 | 公開一覧と申請UIを表示 | 公開一覧とログイン導線を表示 |
| 基本情報 | 表示 | タブ非表示・直接アクセスは会話へ戻る案内 | タブ非表示・直接アクセスは会話へ戻る案内 |

## 公開モデレーター画面

### 常に表示する情報

- 見出し「モデレーター」
- 「モデレーターをしているユーザー」一覧
- 「申請中のユーザー」一覧
- 各ユーザーのニーモニック、完全なユーザーID、IDをコピーする操作
- 申請中ユーザーの申請日時と申請理由
- 読み込み中、空、遅延、失敗を区別する状態メッセージ

### 利用者別の操作

| 利用者 | 表示する操作 | 表示しない操作 |
|---|---|---|
| 未ログイン | 申請位置のログイン | 申請入力、許可・削除選択、直接追加、変更確定 |
| 申請可能な一般ユーザー | 申請理由入力、申請送信 | 許可・削除選択、直接追加、変更確定 |
| 申請中の一般ユーザー | 自身が申請中である表示 | 重複申請、許可・削除選択、直接追加、変更確定 |
| モデレーターまたは会話作成者 | 自身の現在状態 | 自身の申請送信 |
| 会話作成者 | 許可・削除チェックボックス、直接追加、変更の要約、変更確定 | 行ごとの即時許可・即時削除、却下 |

## 会話作成者の一括確定

1. 申請中一覧のチェックは「この申請を許可する」、モデレーター一覧のチェックは「このモデレーターを削除する」と明確にラベル付けする。
2. 選択・直接追加の結果は、確定前に「追加予定」「削除予定」として要約する。
3. 変更がないときは「変更を確定」を無効化する。
4. 確定中はチェックボックス、直接追加、確定操作を無効化し、処理中であることを通知する。
5. 確定時は、現在状態および許可対象申請より新しい発行時刻の一つの会話状態を公開する。
6. 成功時は更新済みの公開一覧を表示し、選択を空にする。失敗時は選択を保持し、再試行可能なエラーを表示する。

## アクセシビリティ

- 各チェックボックスは対象者のニーモニックとユーザーIDを含むアクセシブルな名前を持つ。
- IDコピー操作は、コピー対象を示す名前と成功・失敗の状態通知を持つ。
- 状態メッセージと確定結果は、既存の一つのメインランドマーク内で支援技術に通知する。
- モデレーターと基本情報のタブ追加・非表示後も、矢印、Home、Endによる循環フォーカスと選択状態が正しい順序で機能する。
