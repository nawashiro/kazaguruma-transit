# Data Model: 公開モデレーター管理

## 既存の会話

| 項目 | 意味 | 制約 |
|---|---|---|
| 会話ID | 会話を一意に識別する参照 | 申請とkind 34550の両方で同一であること |
| 会話作成者ID | 会話を作成した利用者 | 基本情報とモデレーター構成を変更できる唯一の利用者 |
| kind 34550発行時刻 | 現在の会話状態の基準時刻 | 同時刻の複数イベントはイベントIDで決定的に一つを選ぶ。変更確定時は選択申請より新しい時刻を用いる |
| モデレーターID集合 | 現在モデレーターをしている利用者 | 重複なし。変更確定により集合全体を更新する |

## モデレーター申請

| 項目 | 意味 | 制約 |
|---|---|---|
| 申請イベントID | 申請を識別する公開ID | 同一時刻の並び順の決定に使用する |
| 申請者ID | 就任を希望する利用者 | 会話ごとに申請中一覧へ最大一人分だけ表示する |
| 対象会話ID | 申請先の会話 | 現在表示中の会話と一致すること |
| 発行時刻 | 申請が公開された時刻 | 現在のkind 34550発行時刻以上のとき申請候補になる |
| 申請理由 | 任意の公開テキスト | 申請中一覧に表示する |

### 導出規則

1. 対象会話を参照し、モデレーター申請として識別される公開イベントだけを候補にする。
2. 候補のうち、現在のkind 34550発行時刻未満のものを除外する。
3. 現在のモデレーターID集合に含まれる申請者を除外する。
4. 申請者ごとに、発行時刻降順・イベントID昇順で最初の一件を「申請中」とする。
5. これにより、各利用者は「モデレーターをしているユーザー」または「申請中のユーザー」の最大一方にのみ現れる。
6. 変更確定時は、現在のkind 34550・許可対象の申請・現在時刻のすべてより後の時刻で新しいkind 34550を発行する。確定前の申請は次の導出で必ず除外される。

## 保留中のモデレーター変更

| 項目 | 意味 | 制約 |
|---|---|---|
| 許可対象ID集合 | 申請中一覧からチェックした利用者 | 現在の申請中利用者だけを選択可能 |
| 削除対象ID集合 | モデレーター一覧からチェックした利用者 | 現在のモデレーターだけを選択可能 |
| 直接追加ID集合 | 会話作成者が入力した利用者 | 有効なユーザーIDであり、現在・許可・削除の集合と重複しない |
| 確定状態 | 未変更、変更あり、確定中、失敗 | 確定成功まで公開の現在状態を変更しない |
| 確定時刻 | 一括更新イベントに用いる発行時刻 | 現在の会話、許可対象申請、現在時刻のいずれよりも後であること |

### 状態遷移

```text
未変更 ──選択・入力──> 変更あり ──確定開始──> 確定中
  ^                         │                    │
  └──選択解除───────────────┘      成功           │失敗
                                  ▼               ▼
                                未変更 <────── 変更あり
```

確定時の次のモデレーター集合は、現在集合から削除対象を除き、許可対象と直接追加対象を加えた集合である。許可と削除の両方に同じ利用者を入れることはできない。
