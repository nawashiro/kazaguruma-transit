# UI Component Boundary Contract

## Purpose

共通UIの責務と、今回の移行範囲を明確にする。009 Nostr read、ルビ外部ライブラリ、PDF生成の内部処理はこの契約の対象外である。

## Button

### Required behavior

- ネイティブのボタン操作、`type`、`disabled`、`aria-*` を明示的に受け取る。
- `loading` 中は操作を受け付けず、処理中状態をアクセシブルに示す。
- `iconOnly` はアクセシブルな名前を必須とする。
- フォーカス表示を消去しない。プロジェクトのDaisyUI規約と44px以上の操作領域を維持する。
- `className` の内容を解析して挙動を変えない。
- 結合表示は `joined` の明示指定で表し、`className` の `join-item` 文字列を解釈しない。
- 日本語の可視ボタン文言は既存の `ruby-text` 子要素規約を保つ。ただし、ボタンの状態管理とルビ外部ライブラリの初期化は混在させない。

### Migration boundary

- 今回変更する `Button` 利用箇所と新規・変更コードはこの契約に従う。
- 全画面の既存直接 `button` を一括移行しない。
- 直接DaisyUI記述を残す場合は、特殊レイアウト、モーダル固有操作、結合・円形などの理由をコードまたは対象計画に残す。

## Input, Tabs, Modal, Error display

- 入力は説明、エラー、required、invalid状態を失わない。
- タブは選択状態、関連パネル、キーボード操作を失わない。
- モーダルは開閉、タイトル、取消、破壊的操作の関係を明確にする。
- エラー表示は利用者向け日本語、再試行または閉じる操作、適切なlive regionを維持する。

## Verification

- 共通UI単体テストで状態、ARIA、フォーカスに関する契約を検証する。
- 代表利用箇所で、結合ボタン、アイコンのみボタン、loading、フォームエラーを確認する。
- 変更対象外の直接記述を見つけても、一括移行せず、後続候補として扱う。
