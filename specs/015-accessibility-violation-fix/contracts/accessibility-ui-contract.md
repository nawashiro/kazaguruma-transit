# Accessibility UI Contract

## Scope

共通レイアウト、共通UI、設定、場所検索、会話作成・編集・管理画面に適用する。これは画面と支援技術の間の受け入れ契約であり、データ保存やAPI契約ではない。

## Control Contract

- ボタン、リンク、入力、タブは目的が分かるアクセシブルな名前を持つ。
- 可視ラベルがある操作は、支援技術に伝わる名前も同じ目的を表す。
- アイコンだけの操作は、対象を含む日本語の名前を持つ。
- 装飾アイコンは読み上げ対象外で、隣接テキストと重複しない。
- 操作領域は 44px 以上で、キーボードフォーカスが視認できる。

## Form Contract

- 入力には可視ラベルを関連付ける。プレースホルダーだけをラベルにしない。
- 必須、入力エラー、説明は入力との関連をプログラムから判定できるようにする。
- エラーは色以外のテキストでも特定でき、修正に役立つ情報を含む。

## Tab and Navigation Contract

- ページ遷移はリンクとして、同一ページ内の表示切替はタブとして表現する。
- タブは選択状態を持ち、タブパネルとの関係とフォーカス順を一貫させる。
- キーボードの矢印、Home、End 操作が必要なタブでは、現在位置と次のフォーカス先が予測可能である。
- 共通ナビゲーションにはスキップ手段があり、メニュー開閉をキーボードで実行できる。

## Status Contract

- 読み込み、検索結果の概要、成功、失敗、エラーは、フォーカスを奪わず必要な範囲で通知する。
- 同じ状態を複数の live region から重複通知しない。
- モーダルなどコンテキストが変わる場合は、名前・説明・フォーカス移動・閉じる操作を提供する。

## Visual Contract

- 通常テキスト、非テキストUI、フォーカス表示は WCAG 2.2 AA のコントラスト基準を満たす。
- 200%文字拡大と狭い画面幅でも、主要操作・フォーム・エラー・状態メッセージは利用可能である。
- 色、アイコン、位置だけを情報の唯一の手段にしない。
