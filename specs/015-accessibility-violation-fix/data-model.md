# Data Model: アクセシビリティ違反の修正

## 永続データ

本機能で新規の永続エンティティ、API payload、SQLite/Prisma schema、Nostrイベントは追加・変更しない。既存の認証、会話、場所、モデレーター情報を表示するためのUI状態だけを扱う。

## UIアクセシビリティ状態

| 状態 | 主な属性 | 遷移 | 利用者への要件 |
|---|---|---|---|
| 操作可能 | enabled / disabled | 入力・権限・処理中状態で変化 | 名前、役割、値、フォーカスを判定できる |
| タブ選択 | selected tab, active panel | キーボードまたはポインターで切替 | 選択状態と表示パネルが関連付く |
| 入力エラー | invalid, error message | 検証失敗時に設定、修正時に解除 | エラーが入力と関連し、色だけに依存しない |
| 処理中 | busy, loading | 操作開始から完了・失敗まで | 操作不能状態と進行状態が通知される |
| 成功・失敗 | status message | 操作結果で更新 | フォーカスを奪わず支援技術に通知される |
| モーダル | open, dialog focus | 開閉時に設定・解除 | 名前、説明、フォーカス移動、閉じる操作を提供する |

## Validation Rules

- すべての操作可能要素は目的を識別できる名前を持つ。
- すべての入力は可視ラベルまたは同等のプログラム上のラベルと説明を持つ。
- タッチ操作の対象は原則として 44px 以上とする。
- 装飾アイコンは支援技術に重複して提示せず、意味を担う情報は日本語テキストで代替する。
- 状態属性は標準の属性名で表し、状態変化は必要な場合だけ status / alert として通知する。
