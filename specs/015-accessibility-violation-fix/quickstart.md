# Quickstart: アクセシビリティ違反の修正

## 前提

- リポジトリルートで作業する。
- `.env.local` と既存の開発環境を用意する。
- キーボード、画面読み上げ、ブラウザの開発者ツールまたはコントラスト測定手段を利用できる状態にする。

## 自動検証

```bash
npm test -- --runInBand
npm run lint
npm run build
```

期待結果:

- 既存テストと追加したアクセシビリティ契約テストがすべて成功する。
- lint と build が成功する。
- 手書きインライン SVG、`area-selected`、44px 未満の対象操作、ラベル未関連付け入力を検出する回帰テストが失敗しない。

実装後の自動検証結果（2026-07-15）:

- `npm test -- --runInBand`: 83 スイート成功、430 テスト成功、17 スキップ。
- `npx tsc --noEmit`: 成功。
- `npm run lint`: 成功。既存コード由来の warning のみ。
- `npm run build`: 成功。Prisma/GTFS の既存ビルド前処理を含む。

## 手動検証シナリオ

1. 設定、場所検索、会話作成、会話編集、会話管理、ログインモーダルを開き、Tab / Shift+Tab だけで主要操作を完了する。
2. サイドバーをキーボードで開閉し、スキップリンクで本文へ移動する。フォーカスが常に視認でき、トラップされないことを確認する。
3. 住所入力、会話タイトル・説明、モデレーター入力で、ラベル、説明、必須、エラー、修正方法が読み上げられることを確認する。
4. ログイン、カテゴリ、承認の切替で、選択状態と表示パネルが読み上げられることを確認する。
5. 検索・保存・読み込み・成功・失敗状態を発生させ、状態メッセージが一度だけ通知されることを確認する。
6. モデレーター削除、閉じる、検索、承認などの操作領域を確認し、44px 未満がないことを確認する。
7. ライト・ダーク両テーマで、通常テキスト、アイコン、境界線、フォーカス表示のコントラストを測定する。
8. ブラウザの文字サイズを 200% にし、狭い画面幅で全対象画面を確認する。横スクロールによる情報・操作の欠落がないことを確認する。

## 実装監査結果

- 対象ソースの手書きインラインSVG: 0件。
- `area-selected`: 0件。`aria-selected` を使用。
- 対象の個別操作は `min-h-[44px]` / `min-w-[44px]` または同等の共通Buttonを使用。
- 住所入力は `住所` ラベルと検索エラーの `aria-describedby` を持つ。
- データモデル、認証フロー、ページ遷移、既存の永続化形式は変更していない。
